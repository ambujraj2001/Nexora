# Nexora - Database Structure

This document outlines the tables, column names, and data types used in the Nexora Supabase/PostgreSQL database.

---

## 1. Table: users
Primary table for user identity and global AI preferences.

| Column | Data Type | Description |
| :--- | :--- | :--- |
| id | uuid | Primary Key |
| access_code | text | Unique user login code |
| full_name | text | User's full name |
| email | text | Unique email address |
| role | text | User's professional role |
| interaction_tone | text | AI tone: professional, casual, etc. |
| response_complexity | integer | Scale of 1-10 for AI response depth |
| voice_model | text | atlas, standard |
| notify_response_alerts| boolean | |
| notify_daily_briefing | boolean | |
| show_demo | boolean | Tracks if onboarding tour completed |
| two_factor_enabled | boolean | |
| is_locked | boolean | Account security status |
| created_at | timestamp | |
| updated_at | timestamp | |

## 2. Table: entries

| Column | Data Type | Description |

## 9. Supporting Tables
- **memory_access**: Tracks usage for "AI Context" transparency.
- **memory_share_codes**: Temporary codes for shared memories.
- **app_members**: Membership registry for shared apps.
- **app_data**: Dynamic storage for app-specific state (JSONB).
- **app_chats**: History of conversations within a specific app.
- **ai_routine_runs**: Logs and results of automated routine executions.
